# Пульт оркестра — описание проекта

## Название

**Пульт оркестра** (`launch-pult`) — Telegram-пульт и CLI для управления запусками Zerocoder поверх оркестра субагентов.

## Проблема

Я веду параллельно несколько запусков — вебинары каждую неделю, акции к датам, интенсивы раз в несколько месяцев. По каждому запуску в проекте лежит файл `_state.md`: этапы конвейера, открытые вопросы, текущая стадия. Проблема в двух местах:

1. **Нет единой картины.** Чтобы понять, что на какой стадии и где я кого-то жду, нужно открывать каждый `_state.md` руками. Статус расползается по папкам.
2. **Оркестр живёт только в Claude Code за компьютером.** Я собрала оркестр из 14 субагентов с knowledge-base и скиллами, но обращаться к нему могу только сидя в редакторе. С телефона — никак.

## Решение

Одна тонкая прослойка, которая делает две вещи:

- **Трекер** проходит по всем `_state.md` в рабочей папке, считает прогресс по чек-боксам, вытаскивает дату обновления, открытые вопросы и текущую стадию — и собирает статус-борд.
- **Мост к оркестру** гоняет любой вопрос через `claude -p` прямо в папке проекта, где лежат субагенты, knowledge-base и скиллы. Оркестр не переписываем — вызываем его в headless-режиме.

Доступ к обоим — через CLI (за компьютером) и через Telegram-бот (с телефона, замкнут строго на мой user ID).

## Целевой пользователь

Я — проджект-менеджер запусков. Один пользователь, приватный инструмент. Бот не публичный: отвечает только на мой Telegram ID, чужие сообщения игнорирует.

## Ценность

- Статус всех запусков — одной командой `/status`, без открывания файлов.
- Доступ к оркестру субагентов с телефона, не привязываясь к редактору.
- Основа для финального проекта: дальше сюда навешивается запуск конкретных субагентов по командам, напоминания по дедлайнам, отчёты.

## Функции MVP

| # | Функция | Где |
|---|---------|-----|
| 1 | Статус-борд по всем `_state.md` (прогресс, дата, открытые вопросы, стадия) | `cli.py status`, `/status` |
| 2 | Вопрос оркестру через `claude -p` | `cli.py ask "..."`, любой текст боту |
| 3 | Замок доступа на один Telegram ID | `src/bot.py` |
| 4 | Парсер устойчив к двум форматам `_state.md` (пусковой конвейер + недельный) | `src/tracker.py` |

## За рамками MVP (следующие шаги)

- Запуск конкретного субагента по команде (`/announce`, `/concept`).
- Напоминания по дедлайнам из `_state.md`.
- Перенос бота на VPS для работы 24/7 (сейчас — пока включён Mac).
- Голосовой ввод вопросов в боте.

## Технологии

- **Python 3.9+** — стандартная библиотека для трекера (без зависимостей).
- **python-telegram-bot 22.5** — асинхронный бот (`ApplicationBuilder`, `run_polling`).
- **Claude Code CLI** (`claude -p`) — headless-вызов оркестра субагентов.

## Архитектура

```
Telegram / терминал
        │
   ┌────┴─────┐
   │  bot.py  │  cli.py        ← интерфейсы (замок на user ID в боте)
   └────┬─────┘──────┐
        │            │
  ┌─────▼─────┐ ┌────▼──────────┐
  │ tracker.py│ │ orchestra.py  │
  │ читает    │ │ claude -p     │
  │ _state.md │ │ в папке проекта
  └───────────┘ └───────┬───────┘
                        │
              оркестр субагентов
              (.claude/agents, knowledge-base, skills)
```

Прослойка тонкая: трекер — чистый парсинг файлов, мост — обёртка над `subprocess`. Вся «тяжёлая» логика остаётся в оркестре, который уже собран.

## Как запустить

```bash
# Трекер — без зависимостей
python3 cli.py status

# Вопрос оркестру (нужен установленный Claude Code)
python3 cli.py ask "Что на очереди по плану?"

# Telegram-бот
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить токен и user ID
python3 src/bot.py
```

## Безопасность

- Токен и user ID — только в `.env`, который в `.gitignore` и в репозиторий не попадает.
- Бот замкнут на один Telegram ID — чужие сообщения отклоняются с записью в лог.
- Оркестр запускается в границах рабочей папки; ничего наружу не отправляется.

## Метрики успеха

- Статус любого запуска виден за одну команду, без открытия файлов.
- Ответ оркестра приходит в Telegram за разумное время (таймаут 180 с).
- Чужой пользователь не может обратиться к боту.
