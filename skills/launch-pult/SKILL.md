# Пульт оркестра — скилл

Тонкая прослойка над оркестром субагентов: трекер статусов запусков + мост к
оркестру через `claude -p`, доступные из CLI и Telegram-бота.

## Когда срабатывает

- «Статус запусков», «что на какой стадии», «где открытые вопросы», «сведи картину».
- «Спроси оркестр…», «ответь через субагентов» — когда нужен headless-вызов оркестра.
- Доработка пульта: парсер `_state.md`, команды бота, новые функции.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `src/tracker.py` | парсер `_state.md` → статус-борд (чистый Python) |
| `src/orchestra.py` | обёртка над `claude -p` в папке проекта (синхронный ответ в чат) |
| `src/montage_jobs.py` | фоновые задачи: ТЗ на монтаж (детерминированный `montage_tz.py`) |
| `src/analysis_jobs.py` | фоновые задачи: сравнение конкурентов + аудит посадки (`claude -p` пишет файл) |
| `src/bot.py` | Telegram-бот, замок на один user ID |
| `mcp-analysis.json` | MCP для фоновых задач: playwright в изолированном headless-профиле |
| `cli.py` | команды `status` и `ask` |
| `skills/launch-pult/SKILL.md` | этот файл |

## Чек-лист: статус запусков

1. Запустить `python3 cli.py status` (или `/status` в боте).
2. Трекер проходит по всем `_state.md` в `WORKSPACE_ROOT` (по умолч. — `Ai-homework/`),
   пропуская файлы со словом `template` и скрытые папки.
3. По каждому запуску: имя папки, заголовок, прогресс по чек-боксам `[x]/[ ]`,
   дата из строки «Обновлено», открытые вопросы, текущая стадия.
4. Если запусков нет — так и сказать, не выдумывать.

## Чек-лист: вопрос оркестру

1. `python3 cli.py ask "вопрос"` или просто текст боту.
2. Под капотом — `claude -p` в `WORKSPACE_ROOT`: подхватываются субагенты,
   knowledge-base и скиллы оркестра.
3. Таймаут — `ORCHESTRA_TIMEOUT` (по умолч. 180 с). Длинный ответ бот режет по 4000 символов.
4. Не выдумывать за оркестр: если `claude` не найден или упал — вернуть ошибку, не подменять ответом по памяти.
5. Бот зовёт `ask_async` (через `asyncio.create_subprocess_exec`) — не висит на тяжёлом
   запросе, параллельно отвечает на другое; сразу шлёт «Принял, работаю…» и держит
   индикатор «печатает», ответ присылает по готовности. CLI зовёт синхронный `ask`.

### Память диалога (только бот)

- Бот держит **сессию на каждый чат**: `ask` зовётся с `session_id` (детерминированный uuid
  из `chat_id` + «эпохи»). Логика — сначала `--resume`, при неудаче `--session-id` (создать).
- Сессии claude хранятся на диске → память переживает перезапуск бота.
- **`/reset`** увеличивает «эпоху» чата → новая сессия, старый контекст забыт.
- CLI (`cli.py ask`) — без session_id, разовый запрос без памяти.

## Чек-лист: фоновые задачи-кнопки (монтаж / конкуренты / аудит)

Тяжёлые задачи не держат диалог: бот ставит **фоновый процесс**, сразу отвечает «считаю
в фоне», а готовый файл досылает наблюдатель (раз в 20–30 с). Источник правды о задаче —
`jobs/<…>/<id>.json`; готовность определяется по появлению файла-результата, поэтому
переживает перезапуск бота.

| Кнопка / команда | Модуль | Начинка | Куда пишет |
|---|---|---|---|
| 🎬 ТЗ на монтаж · `/montage` | `montage_jobs.py` | детерминированный `montage_tz.py` (whisper+OCR) | `agent-assistant/reports/` |
| 📊 Сравнить конкурентов · `/compete` | `analysis_jobs.py` | `claude -p` + скилл `ai-competitors` | `research/` |
| 🔍 Аудит посадки · `/audit` | `analysis_jobs.py` | `claude -p` + субагент `landing-auditor` | `audits/` |

- **Монтаж** ≠ оркестр: это голый скрипт, поэтому отдельный модуль и без `claude -p`.
- **Конкуренты/аудит** = оркестр: фоновый `claude -p` сам пишет файл (Write), готовность по файлу.
- **Изолированный playwright (ключевое).** Фоновый `claude -p` получает `--mcp-config mcp-analysis.json --strict-mcp-config` — playwright там в режиме `--isolated --headless`. Это снимает lock общего профиля с основной сессией: цены/контент за JS берутся прямо с лендингов (🟢), а не из агрегаторов (🟡). Проверено: Нетология (сплошной 403 на WebFetch) → 10 курсов с ценами 🟢 прямо со страниц.
- Права фоновых задач: `Read Write Edit Glob Grep WebFetch WebSearch Task mcp__playwright` — **без `Bash`** (деструктив закрыт).
- Лимит — одна задача анализа за раз (`MAX_CONCURRENT=1`); монтаж считается отдельно.

## Чек-лист: запуск бота

1. `.env` из `.env.example`: `TELEGRAM_BOT_TOKEN` (@BotFather) + `ALLOWED_USER_ID` (@userinfobot).
2. `.env` — только локально, в `.gitignore`. Токен в чат не светить.
3. `pip install -r requirements.txt` в venv, затем `python3 src/bot.py`.
4. Проверить: чужой ID получает «Этот бот приватный», свой — статус и ответы оркестра.

## Права записи

Мост вызывает `claude -p --allowedTools Write Edit` (настройка `ORCHESTRA_ALLOWED_TOOLS`):
- бот **может** создавать и править файлы в рабочей папке («собери анонс и сохрани»);
- бот **не может** удалять — `Bash`/`rm` не разрешён;
- чтение файлов и запуск субагентов доступны и так, по умолчанию.

Проверено в бою: запись проходит, удаление блокируется. Для режима read-only —
очистить `ORCHESTRA_ALLOWED_TOOLS`.

## Границы

- Не переписывать оркестр — пульт только вызывает его в headless-режиме.
- Не коммитить `.env` и токены.
- Бот не рассылает наружу — отвечает только владельцу.
- Запись разрешена точечно (Write/Edit), удаление — нет.
